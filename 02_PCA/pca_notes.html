01
The Big Picture

What problem does PCA solve?

Real-world datasets are high-dimensional — hundreds or thousands of features. The more features you have, the harder everything becomes: models train slower, visualisation is impossible, and performance degrades due to the curse of dimensionality.

PCA answers the question: "Can I represent this data with fewer numbers while keeping the essential story intact?"

Core Definition

PCA is an unsupervised, linear dimensionality reduction technique that transforms the original features into a new set of uncorrelated axes (called principal components) ranked by the amount of variance they capture — most to least.

Key Insight

PCA doesn't select existing features. It creates entirely new features that are linear combinations of the originals. You trade interpretability for compression.

02
Intuition — Build It Visually

Three ways to understand PCA

📷

The Photographer Analogy

A photographer wants to capture a 3D object (your data) in a 2D photo (reduced dimensions). They walk around looking for the angle that reveals the most structure — the side showing the most shape variation. That "best angle" is PC1. The second-best angle (perpendicular to the first) is PC2. PCA finds these angles mathematically.

☁️

The Cloud of Points

Imagine data as a cloud of stars in space. God shines a flashlight on the cloud and casts a shadow on a flat surface. The goal: find the rotation of that flat surface that gives the most spread-out shadow. Maximum spread = maximum variance = maximum information retained. PCA finds that optimal rotation.

🏗️

The Room Analogy (CampusX)

You have features: room size and washroom size. These are correlated (bigger house → bigger both). PCA finds a new axis room' along the direction of maximum spread — this captures the "overall house size" concept. The second axis washroom' (perpendicular) captures the remaining tiny variation. You can drop washroom' safely.

The Core Duality

Maximising variance along a direction is mathematically identical to minimising the perpendicular reconstruction error (the distance from original points to the projected line). This is why PCA is provably optimal for linear compression.

03
Prerequisites — The Math You Need

Variance, Covariance & Eigenstuff

Variance

Measures how spread out a single feature is.

Var(X) = Σ(xᵢ − x̄)² / n

High variance → data is spread wide → more information.

Covariance

Measures how two features move together.

Cov(X,Y) = Σ(xᵢ−x̄)(yᵢ−ȳ) / n

+ve → move together. −ve → move oppositely. 0 → independent.

Covariance Matrix

Generalises to all pairs of features. For p features → p×p symmetric matrix.

C[i,j] = Cov(fᵢ, fⱼ)

Diagonal = variances. Off-diagonal = covariances.

Eigenvectors

For a matrix A, a vector v is an eigenvector if multiplying by A only scales it — never rotates it.

A · v = λ · v

In PCA: eigenvectors of the covariance matrix = directions of principal components.

There are always exactly p eigenvectors for a p×p matrix. They are all mutually orthogonal (perpendicular to each other).

Eigenvalues

The scalar λ that goes with each eigenvector. It tells you how much variance exists along that eigenvector's direction.

λ₁ ≥ λ₂ ≥ ... ≥ λₚ

Sorting eigenvalues in descending order automatically ranks principal components from most to least important.

Explained variance ratio of PCᵢ = λᵢ / Σλ

Why Eigenvectors?

We want the direction w that maximises variance of projected data: max Var(Xw) = max (wᵀ C w) subject to ‖w‖ = 1. Using Lagrange multipliers, the solution is: Cw = λw — the eigenvector equation. So the eigenvectors of the covariance matrix ARE the directions of maximum variance. This isn't an approximation — it's exact.

04
The Full Algorithm

PCA step by step

1
Standardise the data
Subtract mean and divide by standard deviation for every feature.
z = (x − μ) / σ   // mean=0, std=1 after scaling Why this matters: PCA maximises variance. If one feature ranges 0–10,000 and another 0–1, the first will dominate purely because of scale, not because it's actually more informative. Standardisation removes this bias.
2
Compute the Covariance Matrix
C = (1/n) · Xᵀ · X   // X is mean-centred; shape: p×p For a dataset with p features, C is a p×p symmetric matrix. The diagonal tells you variance of each feature; off-diagonals reveal correlations between pairs. This matrix encodes all linear relationships in your data.
3
Eigen Decomposition of Covariance Matrix
Find all eigenvectors and eigenvalues of C. C · vᵢ = λᵢ · vᵢ You get p eigenvectors (each length p) and p eigenvalues. Each eigenvector points in a direction of some variance; the eigenvalue tells you how much variance lies along that direction. In practice sklearn uses SVD (Singular Value Decomposition) instead of explicit eigendecomposition — numerically more stable, same result.
4
Sort eigenvectors by eigenvalue — descending
The eigenvector with the largest eigenvalue becomes PC1, second largest becomes PC2, and so on.
λ₁ ≥ λ₂ ≥ λ₃ ≥ ... ≥ λₚ ≥ 0 PC1 captures the most variance. PC2 is perpendicular to PC1 and captures the next most. All PCs are orthogonal (uncorrelated) by construction.
5
Select top k components
Choose k based on cumulative explained variance (typically ≥ 95%). Explained Variance Ratio of PCᵢ = λᵢ / (λ₁ + λ₂ + ... + λₚ) Form a projection matrix W by stacking the top k eigenvectors as columns: W is shape (p × k).
6
Project the data onto the new subspace
X_reduced = X_scaled · W   // shape: (n × p) · (p × k) = (n × k) The result is your compressed dataset with n samples and only k features (the principal components). These k features are uncorrelated and ordered by importance.
05
Choosing the Right k

How many components to keep?

📊

Scree Plot (Elbow Method)

Plot eigenvalues (or % variance) per PC. Look for the "elbow" — where the line bends sharply. Keep all PCs before the elbow. After the elbow, extra PCs add diminishing information.

📈

Cumulative Variance Threshold

Plot cumulative explained variance. Choose k where it first crosses 95% (general tasks) or 99% (sensitive tasks like medical imaging). In sklearn: n_components=0.95

👁️

Visualisation Goal

If you need to visualise, always use k = 2 (2D plot) or k = 3 (3D plot) regardless of variance percentage — that's the whole point.

Common Mistake

Aiming for 100% cumulative variance means you kept all components — no dimensionality reduction happened at all. Always accept some small loss. Also, don't confuse explained variance (scikit returns ratio, not percentage — multiply by 100).

Implementation

PCA in Python — complete code

import numpy as np import matplotlib.pyplot as plt from sklearn.preprocessing import StandardScaler from sklearn.decomposition import PCA from sklearn.pipeline import Pipeline # ── 1. SCALE (always before PCA) ────────────────────────────────── scaler = StandardScaler() X_train_s = scaler.fit_transform(X_train) # fit + transform train X_test_s = scaler.transform(X_test) # ONLY transform test # ── 2. FIT PCA ──────────────────────────────────────────────────── pca = PCA(n_components=0.95) # keep 95% variance X_train_pca = pca.fit_transform(X_train_s) # fit on train only! X_test_pca = pca.transform(X_test_s) # no fit on test # ── 3. INSPECT ──────────────────────────────────────────────────── print(pca.n_components_) # how many PCs selected print(pca.explained_variance_ratio_) # variance per PC (array) print(pca.explained_variance_ratio_.sum()) # total variance kept # pca.components_ → shape (k, p): the eigenvectors (loadings) # pca.mean_ → mean of training data (used for centering) # ── 4. SCREE PLOT ───────────────────────────────────────────────── plt.figure(figsize=(8, 4)) plt.bar(range(1, len(pca.explained_variance_ratio_)+1), pca.explained_variance_ratio_, color='steelblue', label='Individual') plt.plot(range(1, len(pca.explained_variance_ratio_)+1), np.cumsum(pca.explained_variance_ratio_), 'ro-', label='Cumulative') plt.axhline(y=0.95, color='gray', linestyle='--', label='95% threshold') plt.xlabel('Principal Component'); plt.ylabel('Explained Variance Ratio') plt.legend(); plt.tight_layout(); plt.show() # ── 5. 2D VISUALISATION ─────────────────────────────────────────── pca_2d = PCA(n_components=2) X_2d = pca_2d.fit_transform(X_train_s) plt.scatter(X_2d[:, 0], X_2d[:, 1], c=y_train, cmap='viridis', alpha=0.7) plt.xlabel('PC1'); plt.ylabel('PC2') plt.title('2D PCA projection'); plt.show() # ── 6. PIPELINE (best practice) ─────────────────────────────────── pipe = Pipeline([ ('scaler', StandardScaler()), ('pca', PCA(n_components=0.95)), ('model', LogisticRegression()) ]) pipe.fit(X_train, y_train) pipe.score(X_test, y_test)
Pipeline Best Practice

Always wrap StandardScaler + PCA + Model inside a Pipeline. It prevents data leakage automatically — the pipeline only calls fit on training data and transform on everything else. Use with GridSearchCV to tune n_components as a hyperparameter.

Sklearn PCA Attributes

What every attribute means

AttributeShapeWhat it contains
pca.components_ (k, p) The eigenvectors — each row is one PC direction in original feature space. Also called "loadings". Tells you which original features contribute most to each PC.
pca.explained_variance_ (k,) The eigenvalues — actual variance amounts along each PC. Raw numbers, not ratios.
pca.explained_variance_ratio_ (k,) Eigenvalue of each PC divided by sum of all eigenvalues. Fraction of total variance. Sums to ≤1.
pca.singular_values_ (k,) Singular values from SVD. Related to eigenvalues by: λᵢ = σᵢ² / (n−1).
pca.mean_ (p,) Mean of training data. PCA centres the data before decomposition using this.
pca.n_components_ int Number of components actually kept (useful when you pass a float for n_components).
pca.noise_variance_ float Estimated noise covariance from discarded components (useful for probabilistic PCA).

Geometric Intuition

PCA as a rotation of axes

Think of PCA as rotating the coordinate system to align with the natural axes of your data cloud — not distorting the data itself, just changing your perspective.

Before PCA

Data lives in the original x₁, x₂, ..., xₚ coordinate system. Features are correlated. The "natural shape" of the data cloud is diagonal relative to the axes.

After PCA

Data lives in the PC1, PC2, ..., PCₖ coordinate system. Components are orthogonal (zero correlation). The data cloud's natural elongation IS the axis. You can now drop the short axes safely.

Ellipsoid Interpretation

PCA fits a p-dimensional ellipsoid to the data. Each eigenvector is one axis of the ellipsoid. Each eigenvalue measures the length of that axis. If an axis is very short (tiny eigenvalue), the data barely varies in that direction — drop it. This is fundamentally what dimensionality reduction does.

09
Strengths & Limitations

When PCA shines and when it fails

+Removes multicollinearity — all output PCs are orthogonal (zero correlation)
−Loss of interpretability — PCs are abstract combinations, not original features
+Reduces overfitting by compressing noise into dropped components
−Only linear — can't capture non-linear structures in data
+Speeds up training — fewer dimensions = faster computation
−Sensitive to scale — unscaled data gives wrong results
+Enables 2D/3D visualisation of any high-dim data
−Sensitive to outliers — outliers distort covariance and thus PCs
+Deterministic and computationally efficient (via SVD)
−Information loss is irreversible — you cannot perfectly recover originals
+Always guaranteed to find the globally optimal linear projection
−Doesn't use class labels (unsupervised) — may discard discriminative directions

Under the Hood

SVD — how sklearn actually computes PCA

Why SVD instead of Eigendecomposition?

Computing the full covariance matrix C = XᵀX is expensive and numerically unstable for large p. SVD directly decomposes the data matrix X, is more stable, and handles cases where n < p (more features than samples). Both give identical results.

SVD Decomposition

Any matrix X (n×p) can be written as:

X = U · Σ · Vᵀ

where: U (n×n) = left singular vectors, Σ (n×p) = diagonal matrix of singular values, V (p×p) = right singular vectors (these ARE the eigenvectors of XᵀX, i.e. the principal components).

The principal components are the columns of V. The explained variance = σᵢ² / (n−1) for each singular value σᵢ. Sklearn sets svd_solver='auto' by default — chooses the best algorithm based on data size.

Real-World Applications

Where PCA is actually used

🖼️

Eigenfaces (Face Recognition)

Each face image (e.g., 100×100 pixels = 10,000 features) is compressed using PCA into ~50–100 components called "eigenfaces." New faces are classified by comparing their PC coordinates.

🧬

Genomics / DNA Analysis

Datasets with 500,000+ genetic markers per person. PCA reduces to ~10 components that reveal population structure — literally the geographic origin of ancient human migration patterns.

💹

Finance / Risk Modelling

Hundreds of correlated financial indicators reduced to a few PCs representing systematic risk factors (market risk, interest rate risk). Used in stress testing and portfolio optimisation.

🔊

Signal & Noise Separation

In audio/sensor data, signal lives in top PCs (high variance). Noise is spread across many small PCs (low variance). Reconstructing from top k PCs removes noise automatically.

🏥

Medical Imaging

MRI/CT scan voxel data has millions of features. PCA compresses to manageable representation for anomaly detection, tumour classification, or patient stratification.

🛒

Recommendation Systems

User–item rating matrices are sparse and high-dimensional. PCA (or its cousin SVD) extracts latent factors representing genres, styles, etc. The basis of early Netflix Prize solutions.

Common Mistakes & Pitfalls

What trips people up

MistakeWhy it's wrongCorrect approach
Fitting PCA on test data Leaks test distribution into the model — invalidates evaluation Always fit_transform on train only, transform on test
Skipping standardisation Features with large scales dominate variance calculations unfairly Always apply StandardScaler before PCA
Targeting 100% variance Means no reduction happened — all components kept Target 95–99% based on task sensitivity
Using PCA when non-linear structure exists PCA is blind to curves, spirals, manifolds Use Kernel PCA, t-SNE, or UMAP for non-linear data
Confusing PCA with feature selection PCA creates new abstract features; original ones are gone If interpretability matters, use feature selection instead
Ignoring outliers before PCA Outliers inflate covariance in wrong directions, misaligning PCs Remove/cap outliers before applying PCA
Applying PCA to categorical data PCA assumes continuous numerical input; covariance is meaningless for categories One-hot encode first, or use MCA (Multiple Correspondence Analysis)
14
Points to Remember

Never forget these

★ Exam / Interview Essentials

  • PCA is unsupervised — it has no idea about your labels. It only looks at X, never y.
  • Principal components are orthogonal — they are always uncorrelated by construction. This is mathematically guaranteed.
  • Number of PCs ≤ min(n, p) — you can never get more PCs than features or samples, whichever is smaller.
  • PCA is reversible (approximately) — you can reconstruct originals using pca.inverse_transform(), but with information loss equal to the dropped variance.
  • Scaling is mandatory — without it, PCA rewards units, not information. A feature in kg vs grams would behave completely differently.
  • sklearn uses SVD internally — not explicit eigendecomposition. Same result, better numerical stability.
  • n_components as float (0–1) — tells sklearn the minimum variance to preserve; it auto-selects the number of components needed.
  • Eigenvalue = variance along that PC — directly interpretable. The sum of all eigenvalues = total variance in the scaled data.
  • PCA ≠ Feature Selection — PCA discards original features entirely. Feature selection keeps them. Different goals, different methods.
  • PC1 is the longest axis of the data ellipse — geometrically, the direction of maximum spread. PC2 is the second longest, perpendicular to PC1.
  • Outliers ruin PCA — they pull the covariance matrix and shift the principal directions. Handle outliers before applying PCA.
  • PCA is not always beneficial — if features are already uncorrelated and individually important, PCA may just add noise to the downstream model.
